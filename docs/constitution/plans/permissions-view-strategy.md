# خطة العرض والنطاق (Permissions & View Strategy Plan)

> **الحالة:** Active Draft / Authoritative
> **الغرض:** تثبيت كيف تتعامل الصلاحيات + النطاق + الأدوار لتحديد أي شاشة بيشوفها الموظف.
> **الإصدار:** 1.0
> **التاريخ:** 2026-06-06

---

## 1. المبدأ المعماري (The Core Principle)

### القاعدة الوحيدة:

> **الصلاحية** بتحدد بيأمن ما يقدر يعملو.  
> **النطاق** بحدد اي بيانات بيقدر يشوفو.  
> **الرول** بجمع الإثنين.

ما في النظام "نماون" للفروع الحالي. النظام بياخد قرارين:

```
IF scope = GLOBAL → كل الفروع
IF scope = BRANCH → الفروع الحالي (من الـ dropdown)
IF scope = ASSIGNED → اللي مسند للموظف بس
```

## 2. النطاقات الثلاثة (The Three Scopes)

### 2.1 GLOBAL — كل الفروع

| النطاق | من بيوفو | الفلتر بيالـ API |
|--------|----------|------------------|
| GLOBAL | مدير الشركة | `ما في فلتر` (NO WHERE branch_id) |
| GLOBAL | مدير النظام | `ما في فلتر` |

**مثال:** مدير الشركة بيفتح صفحة الزبائن → بيشوف كل زباين الشركة (فرع 1 + فرع 2 + فرع 3).

**مثال:** مدير الشركة بيفتح صفحة المهام → بيشوف مهام كل الفروع.

### 2.2 BRANCH — الفرع الحالي

| النطاق | من بيوفو | الفلتر بيالـ API |
|--------|----------|------------------|
| BRANCH | مدير الفرع | `WHERE branch_id = actingBranchId` |
| BRANCH | مشرفة | `WHERE branch_id = actingBranchId` |
| BRANCH | فني | `WHERE branch_id = actingBranchId` |
| BRANCH | تيلماركتر | `WHERE branch_id = actingBranchId` |

**مثال:** مدير الفرع بيفتح صفحة الزبائن → بيشوف زباين الفرع الحالي بس.

**مثال:** مدير الفرع بيفتح صفحة المهام → بيشوف مهام الفرع الحالي بس.

#### ما هو الـ "actingBranchId"؟

`الفرع التشغيلي` (الفرع الحالي) = الفرع اللي اختارو من الـ dropdown بالـ Header.

- موظف مربوط بفرع واحد → actingBranchId = هاذاك الفرع.
- موظف مربوط بأكتر من فرع → actingBranchId = الفرع اللي اختارو.

### 2.3 ASSIGNED — اللي مسند للموظف

| النطاق | من بيوفو | الفلتر بيالـ API |
|--------|----------|------------------|
| ASSIGNED | مشرفة | `WHERE client_id IN (اللي مسندين للمشرفة)` |
| ASSIGNED | فني | `WHERE client_id IN (اللي مسندين للفني)` |

**مثال:** المشرفة مسندة للزبون اللي بأعمارهم فارغين. لما بتفتح صفحة الزبائن → بتشوف بس اللي مسندين لها.

**مثال:** الفني مسند للزبون اللي فريقه مكلفهن. لما بيفتح صفحة الزبائن → بيشوف بس اللي مكلفهن لفريقه.

## 3. الشاشات وكيف بتشتغل (Screens & View Logic)

### 3.1 صفحة سجلات الزبائن (Clients List)

| الدور | النطاق | شو بيشوفو |
|-------|--------|------------|
| مدير الشركة | GLOBAL | كل زباين الشركة (فلتر = مفعول) |
| مدير الفرع | BRANCH | زباين الفرع الحالي (فلتر = مفعول) |
| مشرفة | ASSIGNED | الزباين اللي مسندين لها (فلتر = مفعول) |
| فني | ASSIGNED | الزباين اللي مكلفهن لفريقه (فلتر = مفعول) |
| تيلماركتر | BRANCH | زباين كشوف الاتصال (فلتر = مفعول) |

#### ملاحظة هامة على مدير الفرع:

> لما مدير الفرع بيكون مربوط بكل الفروع (multi-branch assignment)، بيقدر يختار أي فرع من الـ **Branch Selector** بالـ Header.
>
> لما بياخد فرع أكتر من فرع → بيشوف زباين هالفرع.
>
> لما بياخد "كل الفروع" من الـ dropdown → بيشوف كل الزباين.
>
> هالشي مش ذات صلاحية — هوي ميزة بيانية (فلترران) مش نطاق.

### 3.2 صفحة المهام (Tasks / Open Tasks)

| الدور | النطاق | شو بيشوفو |
|-------|--------|------------|
| مدير الشركة | GLOBAL | كل المهام (فلتر = مفعول) |
| مدير الفرع | BRANCH | مهام الفرع (فلتر = مفعول) |
| مشرفة | BRANCH | مهام الفرع (فلتر = مفعول) |
| فني | BRANCH | مهام الفرع (فلتر = مفعول) |
| تيلماركتر | BRANCH | مهام كشوف الاتصال (فلتر = مفعول) |

#### ملاحظة:

> لو مدير الفرع بيكون مربوط بكل الفروع → يقدر ياخد مهام كل الفروع من نفس الصفحة.
>
> هوي ميزة ميانية (فلترران) مش نطاق.

### 3.3 صفحة العقود (Contracts)

| الدور | النطاق | شو بيشوفو |
|-------|--------|------------|
| مدير الشركة | GLOBAL | كل العقود |
| مدير الفرع | BRANCH | عقود الفرع |
| مشرفة | ASSIGNED | عقود الزباين المسندين لها |

### 3.4 صفحة الزيارات (Field Visits)

| الدور | النطاق | شو بيشوفو |
|-------|--------|------------|
| مدير الشركة | GLOBAL | كل الزيارات |
| مدير الفرع | BRANCH | زيارات الفرع |
| فني | BRANCH | زيارات فريقه |
| مشرفة | BRANCH | زيارات الفرع |

### 3.5 صفحة الجدولة (Planning / Team Schedule)

| الدور | النطاق | شو بيشوفو |

| فني | BRANCH | جدولة الفرع |
| تيلماركتر | BRANCH | جدولة الفرع |

|-------|--------|------------|
| مدير الفرع | BRANCH | كشوف اتصال الفرع |

## 4. سيناريوهات تفصيلية (Detailed Scenarios)

### سيناريو 1: مدير الفرع و "كل الفروع"

> **المشكلة:** مدير الفرع موكل من وظيفي منها ومن فرع تاني. بياخد مهمة من فرع تانية.

| المشهد | التفصيل |
|--------|--------|
| المهمات | يقدر يشوف مهام كل الفروع (بس ما بيقدر يعدّل) |
| الزبائن | يقدر يشوف زباين كل الفروع (بس ما بيقدر يــعدّل اللي لفرعو) |
| العقود | يقدر يشوف عقود كل الفروع |
| الجدولة | يقدر يشوف جدولة كل الفروع |

**الحل:** من نفس الصفحة، مافي فلتر برانش — هوي فلترران (مياني) مش نطاق.

### سيناريو 2: مدير الفرع و "فرع واحد"

| المشهد | التفصيل |
|--------|--------|
| المهمات | يشوف مهام الفرع بس |
| الزبائن | يشوف زباين الفرع بس |
| العقود | يشوف عقود الفرع بس |
| الجدولة | يشوف جدولة الفرع بس |

**الفلتر بيالـ API:** `WHERE branch_id = actingBranchId`

### سيناريو 3: المشرفة و الزبون المسندين

| المشهد | التفصيل |
|--------|--------|
| الزبائن | بتشوف بس اللي مسندين لها |
| العقود | بتشوف بس عقود اللي مسندين لها |

**الفلتر بيالـ API:** `WHERE client_id IN (SELECT client_id FROM client_assignments WHERE hr_user_id = currentUser)`

### سيناريو 4: الفني و زبون الفريق

| المشهد | التفصيل |
|--------|--------|
| الزيارات | بيشوف زيارات فريقه |
| الزبائن | بيشوف الزباين اللي الفريق المسندين له |

**الفلتر بيالـ API:** `WHERE team_snapshot->>'supervisor_id' = currentUser OR team_snapshot->>'technician_id' = currentUser`

### سيناريو 5: مدير الشركة والإدارات الإدارية

| المشهد | التفصيل |
|--------|--------|
| إدارة الفروع | يقدر يشوف كل الفروع |
| الموظفين | يقدر يشوف كل الموظفين |
| الجداول | يقدر يشوف جدولة كل الفروع |
| القوائم | يقدر يعدّل القوائم |
| **ما** ما يقدر | يعدّل الأدوار، يعدّل الصلاحيات، يشيل الـ Super Admin |

## 5. نموذج الـ UI (UI Patterns)

### 5.1 شكل الـ Header (Branch Selector)

```
├───────────────────────────────────────────────────────────┐
│  🏢 الشركة │ الفرع الرئيسي │ فرع الميدان │ ↓  │ محمد الأحمد │ ▼ │
└───────────────────────────────────────────────────────────┘
        ↓
  [الفرع الرئيسي]
  [فرع الميدان]
  [فرع القائمة]
  [كل الفروع]    ← للمدير الشركة فقط
```

**لمن يشوف هالدروداون:**
- مدير الشركة (مندوب GLOBAL) → بيشوف كل الفروع + اختيار "كل الفروع"
- مدير الفرع (مندوب BRANCH) → بيشوف فرعو بس + فروع اللي مسندين له (من `user_branch_assignments`)
- موظف عادي → فرع واحد فقط

### 5.2 شكل الـ Card / Badge للفلتر

```
┌──────────────────────────────────────────────────────────────────────┐
│  📋 سجلات الزبائن                                │
│──────────────────────────────────────────────────────────────────────├──────────┐
│  🔍 بحث...                                           │  │  │ ▼ │ │
│──────────────────────────────────────────────────────────────────────│ ──────────├──────────┘
│  📎 فلتر بالـ: [الفرع الحالی │ فرع الميدان │ كل الفروع │ ▼ │ │
│──────────────────────────────────────────────────────────────────────│──────────├──────────┘
│  ┌──────────────────────────────────────────────────────────────────────┐ │
│  │ أحمد يوسف                        │ فرع الميدان │ │
│  │ الميدان، شارع التحرير             │ مشتري، مؤهل │ │
│  │ 09xx-xxx-xxx                        │ ● عقد فعال │ │
│  └──────────────────────────────────────────────────────────────────────┘ │
│  ┌──────────────────────────────────────────────────────────────────────┐ │
│  │ محمد أحمد                        │ فرع الرئيسي │ │
│  │ الرئيسي، حي المرجة               │ مؤهل، ليد │ │
│  │ 09xx-xxx-xxx                        │ ○ لا عقد │ │
│  └──────────────────────────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────────────────────┘

        ← لو الفلتر = "كل الفروع" → بيشوف كل الزباين
        ← لو الفلتر = "فرع الميدان" → بيشوف بس زباين الميدان
```

### 5.3 قاعدة الـ Branch Selector

```
الموظف اللي بيدخل → بناءً على:
  1. role_id → من `hr_users`
  2. من `user_branch_assignments` → الفروع المربوطة للموظف
  3. الفروع الافتراضي = أول فرع من الليست
  4. لو في صلاحية GLOBAL → بيظافر اختيار "كل الفروع"
  5. لو ما في صلاحية GLOBAL → بيشوف فروعو بس
```

## 6. الـ API وكيف بيفلتر (API Filtering)

### 6.1 الـ Middleware المسؤول (requirePermission)

```
الـ Backend بيتلقى من الـ JWT:
  - user_id
  - role_id
  - acting_branch_id (من الـ Header أو من الـ Cookie)

الـ middleware بيفحص:
  1. هل الموظف عندو الصلاحية؟ (hasPermission)
  2. ايه نطاقها؟ (getPermissionScope)
  3. لو نطاق = BRANCH → بيولّي `acting_branch_id` للـ query
  4. لو نطاق = ASSIGNED → بيولّي `user_id` للـ query
  5. لو نطاق = GLOBAL → ما بيولّي فلتر
```

### 6.2 مثال على فلتريندق (SQL Example)

```sql
-- لو مدير الفرع بيطلب زبائين
-- ونطاقه = BRANCH
SELECT * FROM clients
WHERE branch_id = :acting_branch_id;

-- لو مدير الشركة بيطلب زبائين
-- ونطاقه = GLOBAL
SELECT * FROM clients;
-- ما في فلتر branch_id!

-- لو المشرفة بيطلب زبائين
-- ونطاقه = ASSIGNED
SELECT * FROM clients c
WHERE c.id IN (
  SELECT client_id FROM client_assignments
  WHERE hr_user_id = :current_user_id
);
```

## 7. التقارير والمالية (Reports & Finance)

### 7.1 من يقدر يشوف التقارير؟

| التقرير | من يشوفو |
|---------|------------|
| تقرير المبيعات (فرع) | مدير الفرع |
| تقرير المبيعات (كل الفروع) | مدير الشركة |
| تقرير المتابعات (فرع) | مدير الفرع |
| تقرير المتابعات (كل الفروع) | مدير الشركة |
| تقرير الفنيين | مدير الفرع |
| تقرير التيلماركتر | مدير الفرع |
| تقرير الالتزامات | مدير الفرع |

### 7.2 صلاحية `clients.installments.view`

| الدور | النطاق | السيناريو |
|-------|--------|--------|
| مدير الفرع | BRANCH | بيشوف الالتزامات للزباين اللي لهم عقود |
| مندوب مبيعات | ASSIGNED | بيشوف التزامات اللي هو مابيعها |
| مدير الشركة | GLOBAL | بيشوف كل الالتزامات |

> **ملاحظة:** هالصلاحية لسا موجودة بـ `permissions` table. لازم تتضاف.

## 8. الفجوات والمخاوف (Edge Cases)

### فجوة 1: موظف ممنوح من فرع واحد

**المشكلة:** موظف منفال من فرع. ما عندو `user_branch_assignments`.

**الحل:** الـ UI بيقدم يفتح فرع أول (من الليست). لو ما في فروع → بيرجع الـ API قيمة `ما في بيانات`.

### فجوة 2: موظف بينتقل لفرع تاني

**المشكلة:** مدير الفرع بيكون منها ومن فرع تاني. بيفتح الـ app من فرع تاني.

**الحل:** الـ `acting_branch_id` بيتغير لما بيغيّر الفرع. الـ API بيرجع البيانات الجديدة.

### فجوة 3: عملية من فرع الي فرع تاني

**المشكلة:** موظف بياخد مهمة من فرع تاني. بيحاول يعديلها.

**الحل:** لو نطاق `tasks.edit` = GLOBAL → يقدر يعدل.  
لو نطاق `tasks.edit` = BRANCH → يقدر يعدل بس لو كان الفرع هوي نفسه.

## 9. ملخص الـ Decisions

### DEC-001: لماذا ما في دور "مدير فرع جملي"؟

**القرار:** النظام ما بيفرق بين مدير الفرع ومدير الشركة.  
مدير الفرع مندوب BRANCH — ما يقدر يولّي نفسو لـ GLOBAL.  
لو الشركة بتريد مكلف بيشوف كل الفروع بس ما يقدر يعدل النظام → بينشئو `company_manager`.

### DEC-002: لماذا "مشرفة" ASSIGNED وما "فني" ASSIGNED؟

**القرار:** المشرفة بتتعامل مع الزباين اللي مسندين لها شخصياً (من قبل `client_assignments`).  
الفني بيتعامل مع الزباين اللي فريقه مكلفهن (من قبل `team_snapshot`).

### DEC-003: لماذا لما في صلاحية `clients.installments.view`؟

**القرار:** في النظام كل موظف يقدر يشوف الالتزامات للزبون اللي لهم علاقة مباشرة.  
لما لما نفولو رلي للمالية → لازم نساند هالصلاحية.
