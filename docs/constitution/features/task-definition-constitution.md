# دستور تعريف المهمة (Task Definition Constitution)

لكل مهمة جديدة (أو موجودة بدنا نثبتها)، لازم نحدد:

| # | العنصر | الوصف |
|---|--------|-------|
| 1 | الصلاحية للإنشاء | مين له صلاحية يخلق المهمة؟ (مشرف، تقني، محاسب، تلقائي...) |
| 2 | آلية الإنشاء | يدوي / تلقائي / كلاهما |
| 3 | مودل الإنشاء | شو الحقول يلي بتدخل وقت الإنشاء؟ من وين بتجي كل معلومة؟ |
| 4 | الحالات (Status Lifecycle) | شو حالات المهمة؟ كيف بتنتقل من واحدة للتانية؟ |
| 5 | مين يسجل النتيجة | نفس المنشئ؟ فريق؟ مشرف؟ |
| 6 | مودل تسجيل النتيجة | شو الحقول يلي بتتسجل وقت التنفيذ؟ |
| 7 | نتائج المهمة (Task Outcomes) | شو النتائج الممكنة؟ (نجاح، فشل، مؤجل، جزئي...) |
| 8 | معلومات المهمة | وصف، أولوية، نوع، مدة متوقعة... |
| 9 | العلاقات (Relations) | بتتعلق بأي كيانات؟ (عقد، زبون، جهاز، فرع...) |
| 10 | العنوان المكاني (Zone/Location) | بتستخدم عنوان الزبون ولا عنوان العقد؟ |
| 11 | التاريخ المرجعي | بتستخدم تاريخ الاستحقاق ولا الموعد المتوقع؟ |
| 12 | شروط الإنشاء (Validation) | شو الشروط يلي لازم تتحقق قبل الإنشاء؟ |
| 13 | التأثيرات الجانبية (Side Effects) | لما تكتمل، شو بيصير بكيانات تانية؟ (جهاز بيغيّر phase، عقد بيتحدث...) |
| 14 | الـ Permissions | بدها صلاحيات جديدة ولا بنستخدم الموجودة؟ |
| 15 | الـ UI/Forms | شو الحقول يلي بتظهر بالفورم؟ |
