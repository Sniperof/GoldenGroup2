# دستور الفيتشر — الإدخال اليدوي لطلب التوظيف

> الحالة: معتمد كفيتشر دستوري للدومين
> المصدر: `packages/web/src/pages/jobs/ManualApplicationEntry.tsx`، `packages/api/routes/adminApplications.ts`

## 1) الملخص التنفيذي
الإدخال اليدوي هو مسار إداري لإنشاء نفس كيان الطلب عبر الموظف بدلًا من التقديم العام.

## 2) العقد التشغيلي
### 2.1 حدود الفيتشر
- يشمل نموذج الإدخال الإداري ومسار الإنشاء فقط.
- لا يعرّف كيانًا جديدًا.
- ينتج نفس الطلب المستخدم في بقية الدومين.
- يعيد استخدام قواعد التحقق والربط والانتقال نفسها.

### 2.2 المسار التشغيلي
1. فتح نموذج الإدخال اليدوي.
2. اختيار الشاغر أو متابعة المسار الإداري المسموح.
3. إدخال بيانات المرشح.
4. إضافة بيانات الوسيط فقط إذا كان نوع الإدخال يحتاج ذلك.
5. حفظ الطلب.
6. متابعة المسار نفسه المستخدم لأي طلب آخر.

### 2.3 القواعد التشغيلية
- الإدخال اليدوي يجب أن ينشئ نفس كيان الطلب.
- الذي يتغيّر هو مصدر الإدخال فقط، لا دورة حياة الطلب.
- يستخدم نفس قواعد ربط الشاغر وتصفية الأهلية.
- مسار `Refer a Candidate` يفعّل متطلبات الوسيط.
- أي شرط بيانات مطلوب في النموذج العام يبقى مطلوبًا عند تطبيقه هنا.
- ضمن عنوان السكن، المحافظة والعنوان التفصيلي فقط إلزاميان؛ المنطقة والناحية والحي اختيارية.
- ضمن المؤهلات والخبرات، الحقل "هل تمتلك سيارة" إلزامي ويُحفظ كقيمة نعم/لا.

### 2.4 الصلاحيات والنطاق
- `jobs.applications.create`
- الصلاحية تعمل ضمن نطاق `GLOBAL` أو `BRANCH` بحسب السياق.

### 2.5 حالات الحافة
- أي إدخال يدوي يفتقد الحقول المطلوبة يجب أن يُرفض.
- إذا كان وضع الإحالة مفعلًا، يصبح قسم الوسيط إلزاميًا.
- الشاغر المختار، إن استُخدم، يجب أن يكون مؤهلًا.
- لا يجوز للنموذج أن ينشئ نموذج تخزين موازيًا أو دورة حياة موازية.

### 2.6 التحقق
- التأكد أن الحفظ اليدوي ينشئ سجل طلب طبيعيًا.
- التأكد أن صفحة التفاصيل اللاحقة تعرض نفس السجل ونفس دورة الحياة.
- التأكد أن الحفظ لا يفشل عند ترك المنطقة أو الناحية أو الحي فارغة.
- التأكد أن حقول الوسيط تظهر فقط عند اختيار وضع الإحالة.
- التأكد أن "هل تمتلك سيارة" لا يمكن تركه فارغًا وأن قيمته تظهر في تفاصيل الطلب.

## 3) العقد التقني
### 3.1 الكيانات والحقول
يعيد استخدام حقول المرشح والوسيط والطلب من فيتشر `طلبات التوظيف`.

### 3.2 عقد الواجهة
- `packages/web/src/pages/jobs/ManualApplicationEntry.tsx`
- يجب أن تطابق التحققات قواعد السجل المخزن.

### 3.3 عقد الـ API
- `POST /api/admin/applications`
- نفس مسار الإنشاء المستخدم في الإدخال الإداري.

### 3.4 التوافق الخلفي
- الإدخال اليدوي يبقى مسارًا وليس نموذج تخزين مستقلًا.
