# Gaps And Questions

## الهدف

هذا الملف مخصص لتجميع النقاط التي ما تزال غير محسومة أو المختلطة بين:

- التوثيق
- الكود
- المعنى التشغيلي

> ملاحظة: القرارات التي حُسمت في جلسة المقابلة مع صاحب المنتج موثقة في [`08-resolved-decisions.md`](./08-resolved-decisions.md).
> هذا الملف يحتفظ فقط بالفجوات **التي ما زالت مفتوحة** بعد تلك الجلسة.

## فجوات مفهومية محسومة (تحويل مرجعي)

| الفجوة | حُسمت في |
|---|---|
| العقد يبدو كأنه يملك الجهاز كاملاً | DEC-CT-09 (سجل الحيازة) + DEC-CT-02 (فصل service_agreements) |
| تطابق `paymentType` و `method` | يُعاد فحصه بعد تطبيق DEC-CT-06 + DEC-CT-08 |
| الفرق بين القسط والمستحق | DEC-CT-06 (الذمة = رصيد محسوب) |
| ملكية البيعة مقابل منشئ العقد | DEC-CT-11 (`sale_owner_id`) + DEC-CT-12 (`collection_owner_id`) |
| الحدود بين contracts / installed_devices / device_warranties | DEC-CT-04 + DEC-CT-05 + DEC-CT-09 |
| المصدر الرسمي لحالة الجهاز | DEC-CT-03 (قاموس موحد + migration) |
| قواعد إنشاء dues من الأقساط | DEC-CT-06 (لا dues — رصيد محسوب) |
| ContractDetail: بيانات عقد vs جهاز | DEC-CT-09 + DEC-CT-02 (الفصل واضح) |

## فجوات لا تزال مفتوحة

### 1. تفاصيل migration قاعدة البيانات

القرارات الجوهرية محسومة لكن تنفيذ migration يحتاج حسم:

- ترتيب تنفيذ migrations: حالات أولاً أم جداول جديدة أولاً؟
- استراتيجية backward compatibility خلال فترة التحول.
- توافق `dues` API الحالي مع طبقة عرض جديدة محسوبة من الأقساط.

### 2. صيغة `paymentType` على العقد مقابل `method` على الدفعة

بعد DEC-CT-06 و DEC-CT-08، يلزم مراجعة:

- هل `paymentType` على العقد ما زال له معنى؟ أم يكفي توزيع الدفعات + entry_type؟
- توحيد الـ enums بين `shared/types.ts` والـ API.

### 3. تمثيل `amendment` للعقد القابل للطباعة

DEC-CT-15 ذكر `amendment` لكن دون تفصيل:

- ما الذي يستوجب إصدار amendment تلقائي؟
- هل تُعرض جميع النسخ في تاريخ العقد، أم النسخة الأخيرة فقط؟
- علاقة amendment بتغيير الأقساط/الكفالة/الأطراف.

### 4. سيناريوهات حيازة لم تُغطَّ صراحةً

DEC-CT-09 ثبّت بنية `device_possession_log`، لكن سيناريوهات حافة تحتاج توثيقاً:

- الجهاز عند الفني خلال زيارة قصيرة (هل يفتح صف؟)
- الجهاز البديل المؤقت (swap) — صفّان متوازيان أم تسلسل؟
- استرجاع جهاز بعد إلغاء — هل يفتح صف جديد للمخزن؟

### 5. علاقة `service_agreements` بـ `installed_devices`

DEC-CT-02 فصل عقد الصيانة، لكن:

- هل الجهاز الخارجي يُسجّل في `installed_devices` رغم عدم بيعه؟
- أم في جدول مستقل `external_devices` يرتبط بـ `service_agreements`؟
