# البرومت النهائي — مهمة تسليم الجهاز (device_delivery)

> **ملاحظة حاسمة:** كل ما يظهر للموظف بالواجهة يكون بالعربية فقط. لا يظهر أي مصطلح إنجليزي أمام المستخدم.
> **النطاق:** staging فقط.

---

## ١. تعريف المهمة

**نوع المهمة:** تسليم جهاز
**العائلة:** توصيل وتركيب
**إلزامية العقد:** نعم — لا يمكن إنشاؤها بدون عقد
**تاريخ الاستحقاق:** لا يُستخدم — نستخدم فقط "التاريخ المتوقع"

---

## ٢. إنشاء المهمة

### ٢.١ إنشاء تلقائي (موجود حالياً)

عند حفظ عقد بيع جديد، ينشئ النظام مهمة تسليم تلقائياً:
- الزبون ← من العقد
- الجهاز ← من العقد
- عنوان التسليم ← عنوان التركيب المسجل بالعقد
- التاريخ المتوقع ← تاريخ التسليم المحدد بالعقد، أو بعد ٣ أيام
- السبب ← "إنشاء من عقد"
- المصدر ← النظام

### ٢.٢ إنشاء يدوي (جديد — يضاف)

**أين يظهر زر الإنشاء:**
صفحة الزبون → تبويب "العقود" → كرت تتبع الجهاز

**شروط ظهور الزر:**
- العقد نشط
- حالة الجهاز بالعقد: "قيد التسليم"
- لا توجد مهمة تسليم مفتوحة لنفس العقد

**الحقول التي يملأها الموظف:**

| الحقل | إلزامي | الوصف |
|-------|--------|-------|
| العقد | نعم | قائمة منسدلة — العقود النشطة يلي جهازها "قيد التسليم" |
| اسم الزبون | نعم | تلقائي من العقد — غير قابل للتعديل |
| نوع الجهاز | نعم | تلقائي من العقد — غير قابل للتعديل |
| عنوان التسليم | نعم | افتراضي = عنوان التركيب من العقد — قابل للتعديل |
| التاريخ المتوقع | نعم | بيحدده الموظف مع الزبون |
| السبب | نعم | قائمة منسدلة من "إدارة القوائم" — نوع "أسباب المهام" |
| الأولوية | لا | عالية / متوسطة / منخفضة |
| ملاحظات | لا | نص حر |

**ما لا يُطلب عند الإنشاء:**
- لا تاريخ استحقاق — مخصص للذمم فقط
- لا فريق تنفيذ — بيتعين بمرحلة الجدولة
- لا رقم تسلسلي — الفني بيسجله عند التسليم

---

## ٣. تتبع الجهاز بالواجهة (ClientProfile → العقود)

خطوات العمل تظهر للموظف بالشكل التالي:

### عند "قيد التسليم"
```
[تسليم الجهاز] ← نشطة الآن
[تركيب الجهاز] ← مخفية
[تشغيل الجهاز] ← مخفية
زر: "إضافة مهمة تسليم"
```

### عند "تم التسليم"
```
[تسليم الجهاز] ← مكتملة ✓
[تركيب الجهاز] ← نشطة الآن
[تشغيل الجهاز] ← مخفية
زر: "إضافة مهمة تركيب"
```

### عند "تم التركيب"
```
[تسليم الجهاز] ← مكتملة ✓
[تركيب الجهاز] ← مكتملة ✓
[تشغيل الجهاز] ← نشطة الآن
زر: "إضافة مهمة تشغيل"
```

## ٥. صفحة تفاصيل المهمة

**المسار:** /مهام/تسليم/[رقم]

**الأيقونة:** شاحنة
**لون الأيقونة:** سماوي

### التبويبات الستة:

١. **نظرة عامة**
   - ملخص المهمة (حالة، أولوية، سبب)
   - الجدولة (التاريخ المتوقع + تاريخ/وقت الزيارة + نوع المهمة)
   - بيانات الإنشاء + إحصائيات
   - تنبيهات إن وجدت

٢. **بيانات الزبون**
   - الاسم الكامل + الهاتف + العنوان

٣. **العقد والجهاز**
   - رقم العقد + تاريخه
   - نوع الجهاز + الرقم التسلسلي (إن وجد)
   - المالية (نوع الدفع + السعر)

٤. **التواصل والمتابعة**
   - اتصالات التيلماركتينج
   - سجل الأحداث
   - إضافة ملاحظة

٥. **معلومات التركيب** ← تبويب إضافي
   - إذا كانت المهمة "تسليم": بيانات التسليم (الرقم التسلسلي، العنوان، الحالة...)
   - إذا كانت المهمة "تركيب": بيانات التركيب (مصدر المياه، التمديدات...)
   - إذا كانت المهمة "تشغيل": بيانات التشغيل (TDS، اختبار الضغط...)

٦. **النتيجة**
   - عرض النتيجة المسجلة إن وجدت
   - أو زر "تسجيل النتيجة"

## ٦. تسجيل النتيجة

### الحقول التي يسجلها الفني:

| الحقل | إلزامي | الوصف |
|-------|--------|-------|
| النتيجة | نعم | نجاح / العميل غير متوفر / عنوان خاطئ / رفض الاستلام |
| الرقم التسلسلي | نعم (إن نجح) | ما هو مكتوب على الجهاز |
| حالة الجهاز عند الاستلام | نعم (إن نجح) | سليم / ضرر طفيف / نقص ملحقات |
| عنوان التسليم الفعلي | لا | إن اختلف |
| تاريخ التسليم الفعلي | لا | افتراضياً اليوم |
| مَن سلّم؟ | لا | الفني المسجل بالزيارة |
| إقرار الزبون | لا | توقيع أو تأكيد |
| صور الجهاز | لا | مصفوفة صور |
| ملاحظات | لا | سبب الرفض أو ملاحظات إضافية |

### تأثير كل نتيجة:

| النتيجة | حالة المهمة | حالة الجهاز | ما يحدث بعدها |
|---------|------------|-------------|---------------|
| تم التسليم بنجاح | مكتملة | تم التسليم | ينشئ تلقائياً مهمة تركيب |
| العميل غير متوفر | مكتملة | قيد التسليم | ينشئ مهمة تسليم جديدة (متابعة) |
| عنوان خاطئ | مكتملة | قيد التسليم | ينشئ مهمة تسليم جديدة + تحديث العنوان |
| رفض الاستلام | مكتملة | قيد التسليم | قرار المشرف: إلغاء أو متابعة |

> ملاحظة: المهمة الأصلية تصبح "مكتملة" دائماً — النتيجة سجل محاولة. إن فشلت، تنشأ مهمة جديدة.

## ٧. قاعدة البيانات

### ٧.١ الجدول الأساسي (open_tasks)

```sql
الزبون             ← من العقد
الفرع             ← من العقد أو المستخدم
رقم_العقد         ← من العقد

نوع_المهمة        = 'device_delivery'
عائلة_المهمة      = 'delivery'
السبب             ← من إدارة القوائم
الحالة            = 'open'

التاريخ_المتوقع   ← الموظف
تاريخ_الاستحقاق   = NULL  ← لا يُستخدم

الأولوية          ← الموظف
ملاحظات           ← الموظف

المصدر            = 'manual' أو 'system'
منشئ_المهمة       ← المستخدم الحالي
```

### ٧.٢ جدول نتيجة التسليم (visit_task_device_delivery_results)

```sql
رقم_نتيجة_المهمة   ← رقم السجل العام
الرقم_التسلسلي     ← الفني بيقرأه
رقم_موديل_الجهاز   ← تأكيد

عنوان_التسليم      ← ممكن يختلف
تاريخ_التسليم_الفعلي ← تاريخ التنفيذ

مسلّم_بواسطة      ← الفني المسجل
إقرار_الزبون      ← نعم/لا

حالة_الجهاز       = سليم / ضرر_طفيف / نقص_ملحقات
صور_التسليم       ← مصفوفة روابط
```
