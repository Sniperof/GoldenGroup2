# PROMPT: تصميم صفحة تفاصيل العقد (Contract Detail Page) — UI/UX احترافي

## الهدف
رسم وتنفيذ صفحة تفاصيل العقد (`/contracts/:id`) بشكل احترافي، منظّم، ومرتّب على مجموعات مفهومة.

**القاعدة:** كل حقل من `contracts` لازم يظهر بمجموعتو المنطقية. لا حقول مبعثرة.

---

## 🎨 التصميم العام

- **لون الخلفية:** `bg-slate-50` — رمادي فاتح جداً
- **البطاقات (Cards):** `bg-white` + `rounded-2xl` + `shadow-sm` + `border border-gray-100`
- **الهيدر:** sticky top + gradient خفيف sky-blue
- **الخطوط:** Arabic-compatible font (Noto Sans Arabic or similar)
- **Western numerals فقط** — ممنوع الأرقام الهندية
- **الاتجاه:** RTL بالكامل

**قواعد:**
- **الهيدر** = MiniSnapshot: `[avatar] الاسم الكامل (اللقب) [classification]` + `primaryMobile` + `addressShort`
- **"العنوان الكامل"** = Standard Snapshot: 4 مستويات geo + `detailedAddress` + خريطة
- **المهنة + مهنة الزوج/ة** دايماً تظهر — "غير محدد" إذا فاضي
- **"الهوية القانونية"** = قسم منفصل تحت خط فاصل — `buyer_national_id`, `buyer_gender`, `buyer_birth_date`, `buyer_mother_name`, `buyer_national_id_*`
- **الملكية** = `ownershipDisplay` + `sourceChannel`
- **الأرقام** Western numerals فقط

**⚠️ ملاحظة مهمة لـ المطور:**

حقل `buyer_national_id` (رقم الهوية الوطنية نفسه) **غير موجود حالياً** بجدول `contracts`. الموجود بس:
- `buyer_national_id_registry`
- `buyer_national_id_issued_by`
- `buyer_national_id_issue_date`
- `buyer_national_id_box`

**الحل المطلوب:**
1. أضف `ALTER TABLE contracts ADD COLUMN buyer_national_id VARCHAR(50);` للـ migration
2. أو: إذا المشتري = الزبون (`customer_id`)، اسحب `national_id` من `clients`
3. أو: إذا المشتري مختلف عن الزبون، الحقل بيفضل فارغ لحد ما يملوه يدوياً

## ⚠️ قواعد صارمة

1. **Western numerals فقط** — ممنوع الأرقام الهندية
2. **التواريخ:** YYYY-MM-DD (مثلاً 2026-05-20)
3. **الأسعار:** بفواصل آلية + "ل.س" (مثلاً 2,250,000 ل.س)
4. **المهنة + مهنة الزوج/ة:** دايماً تظهر — "غير محدد" إذا فاضي
5. **العنوان:** 4 مستويات geo إلزامية + `detailedAddress`
6. **الأرقام القانونية:** Western numerals (رقم السجل المدني، الصندوق...)
7. **الروابط:** كل ID معرّف → رابط لصفحتو
8. **الحالة (status):** badge ملوّن دايماً ظاهر
9. **المسؤول/ين:** ownershipDisplay = اسم أول مسؤول + "+N" أو اسم الفرع
10. **الـ classification:** badge جانب الاسم (LEAD/OP/FOP)
