### السبب الجذري

المشكلة **ليست** في صلاحيات الـ geo endpoint — فهو مفتوح لجميع المستخدمين المسجّلين بدون قيود. المشكلة في **`Clients.tsx`**:

```typescript
// Clients.tsx — lines 44-65
const [clientsData, visitsData, contractsData, geoUnitsData] = await Promise.all([
    api.clients.list(),    // ← قد تفشل (403) إذا المشرفة ليس لديها صلاحية
    api.visits.list(),     // ← قد تفشل (403)
    api.contracts.list(),  // ← قد تفشل (403)
    api.geoUnits.list(),   // ← تنجح دائماً — لكنها مربوطة بالباقي
]);
setGeoUnits(geoUnitsData); // ← لا يُنفَّذ أبداً إذا فشلت أي call أعلاه
```

`Promise.all` — إذا فشلت **أي** call واحدة، يقع الكل في `catch` ولا يتم `setGeoUnits` إطلاقاً → `geoUnits = []` → GeoSmartSearch لا يعرض شيئاً.

---

### لماذا مدير الفرع يرى المناطق؟
لديه صلاحيات `clients.view_list` + `visits.view_list` + `contracts.view_list` → `Promise.all` ينجح → `geoUnits` تُحمَّل ✓

### لماذا المشرفة لا تراها؟
على الأرجح تملك `clients.create` لكن ليس لديها `visits.view_list` أو `contracts.view_list` → call واحدة تُرجع 403 → `Promise.all` يسقط → `setGeoUnits` لا يُستدعى → المناطق لا تظهر ✗

### جدول الـ Endpoints

| Endpoint | صلاحية مطلوبة | المشرفة تملكها؟ |
|----------|--------------|----------------|
| `GET /geo-units` | لا شيء (public) | ✓ |
| `GET /system-lists` | `requireAuth` فقط | ✓ |
| `GET /clients` | `clients.view_list` | ✓ غالباً |
| `GET /visits` | ؟ | **مجهول** |
| `GET /contracts` | ؟ | **مجهول** |
